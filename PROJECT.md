# フォトナHub

## ミッション
Fortniteプレイヤーが毎日最初に見るサイト

## 優先順位
1. 公開速度
2. ユーザー価値
3. 保守性
4. 拡張性

## 禁止
- AI機能
- 会員機能
- コメント
- 通知
- 複雑なバックエンド

## 技術方針
- シンプルに作る
- ライブラリ追加は最小限
- 既存コードを優先利用
- Vercelで動くことを前提

---

## 作業ルール（厳守）

複雑な設計を提案しない  
今必要ない機能を提案しない  
まず既存コードを理解する  
変更前に影響範囲を説明する  
大規模リファクタリング禁止  
ユーザー価値を優先する  
公開を遅らせる提案は禁止  
**作業完了のたびに TASKS.md を更新する（完了タスクに ✅・日付・内容を反映）**  
**TASKS.md 更新後・main push後は必ず Vercel デプロイ状況を確認してエラーがないかチェックする**  

## デプロイ手順（厳守）

本番への直接デプロイ禁止。必ず以下の手順を踏む。

```
1. develop ブランチで作業・変更
2. git push origin develop
   → Vercel がプレビューURL を自動生成
3. しゅうやがプレビューURLで確認
4. OK → main にマージ → 本番に自動反映
   NG → develop で修正してプレビューを再確認
```

| ブランチ | 用途 | URL |
|---------|------|-----|
| develop | 確認用（プレビュー） | Vercelが自動生成するURL |
| main | 本番 | fortnite-hub-delta.vercel.app |

## 作業前に毎回出力すること

コードを書く前に以下を出力する。

1. 現状理解
2. 問題点
3. 解決方法
4. 変更ファイル
5. リスク

理解できたら実装を開始する。
